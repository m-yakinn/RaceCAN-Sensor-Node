# Future Roadmap

This roadmap describes the planned development path for RaceCAN Digital Kit after Week 1.

## Phase 1: Software Simulation

Status: In progress

Goals:

1. Build telemetry simulator
2. Build terminal dashboard
3. Add CSV logging
4. Add command line options
5. Document system behavior

## Phase 2: Better Dashboard

Goals:

1. Add cleaner terminal formatting
2. Add fault history
3. Add summary statistics
4. Add log file selection
5. Add optional Streamlit dashboard
6. Add plots for voltage, temperature, and current

## Phase 3: Firmware Template

Goals:

1. Add Arduino style firmware template
2. Add STM32 style firmware notes
3. Add sensor reading functions
4. Add fault logic functions
5. Add CAN transmit pseudocode
6. Add firmware architecture documentation

## Phase 4: Hardware Design

Goals:

1. Create KiCad schematic
2. Design sensor input circuits
3. Add CAN transceiver circuit
4. Add voltage regulation
5. Add reverse polarity protection
6. Add TVS diode protection
7. Add test points
8. Create PCB layout
9. Export Gerbers
10. Create BOM

## Phase 5: Digital Product Packaging

Goals:

1. Create setup guide
2. Create user manual
3. Create design explanation PDF
4. Add example logs
5. Add screenshots
6. Add product landing page
7. Package files as downloadable digital kit

## Phase 6: Optional Physical Validation

Goals:

1. Build breadboard prototype
2. Test sensor readings
3. Test CAN communication
4. Order PCB
5. Assemble board
6. Validate with oscilloscope or logic analyzer

## Long Term Vision

The long term goal is to create a complete digital engineering kit that teaches students how to understand, simulate, design, and eventually manufacture a CAN based low voltage telemetry node.
